## Вариант переноса проекта из оффлайн в онлайн

Бот лучше сразу создавать онлайн. Нужен хороший тектовый редактор в онлайне.  

> Сделать git init в проекте и создать онлайн-репозиторий

> Создать поддомен (или использовать технический поддомен)

> Создать SSL-сертификат к поддомену

> Если бот необходимо скачать из онлайн репозитория в папку поддомена, то скачать по SSH файлы бота c онлайн-репозитория в корень поддомена (либо как файлы, либо как папку с файлами)

    git clone https://github.com/user/myrepository

## Установить webhook для бота в адресной строке браузера.

Документация: https://core.telegram.org/bots/api#setwebhook

> Кратко:

Установить Вебхук

    https://api.telegram.org/botYOURTOKEN/setWebhook?url=https://mysub.domen.ru/bot/index.php

Посмотреть информацию по Вебхуку

    https://api.telegram.org/botYOURTOKEN/getWebhookInfo

В адресной строке надо указывать слово bot и далее свой токен, это всё слитно - botYOURTOKEN.

> Подробно:

Вероятно, если в корень поддомена (домена) положена папка, внутри которой файл index.php, то целью указывается этот файл index.php.
Предпочёл этот вариант, а не следующмй с указанием корня поддомена. Потому что к работающему поддомену можно получить доступ по https в браузере. Тогда в брузер будет загружен index.php бота. А это кажется нежелательным.

    https://api.telegram.org/botYOURTOKEN/setWebhook?url=https://mysub.domen.ru/bot/index.php

Или, если в корень поддомена (домена) положены файлы наряду с файлом index.php, то целью указывается этот файл index.php или сам поддомен (домен):

    https://api.telegram.org/botYOURTOKEN/setWebhook?url=https://mysub.domen.ru/index.php

    https://api.telegram.org/botYOURTOKEN/setWebhook?url=https://mysub.domen.ru/

Также - не обязательно использовать поддомен, можно домен.

Ответ должен быть:

    {"ok":true,
     "result":true,
     "description":"Webhook was set"
    }

Если webhook установлен {"ok":true, ...}, то можно отправлять сообщения боту из Телеграм.

Проверить статус бота на webhook можно так

    https://api.telegram.org/botYOURTOKEN/getWebhookInfo

Должен быть ответ

    {"ok":true,
        "result": {
            "url":"https://..../bot/index.php",
            "has_custom_certificate":false,
            "pending_update_count":0,
            "max_connections":40,
            "ip_address":"...."
        }
    }

## Переустановка бота на другой домен

Если нужно переустановить бота на другой сайт, надо сразу использовать setWebhook с новым адресом, нет смысла удалять или отвязывать Вебхук от старого домена.
