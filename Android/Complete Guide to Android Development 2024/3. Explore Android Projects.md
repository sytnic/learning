## 018-Explore the project structure

Некоторые папки (обозначены зелёным квадратиком) являются модулями. Модули - это исходные файлы и файлы настроек, которые разделены по функциональности.  

Типы модулей:

- App module
- Dynamic feature module
- Library module
- Automotive module 

## 020-Manage Gradle build scripts

Файлы сборки называются `.gradle` для Groovy и `.gradle.kts` для Kotlin. Эти файлы есть для каждого модуля и отдельно для всего проекта в целом.  

Файл сборки всего проекта `/settings.gradle.kts` - содержит список репозиториев, где Gradle будет искать зависимости, название приложения и список его модулей.  
Репозиториями могут быть, например, `google, mavenCentral, gradlePluginPortal`.  
В этом файле также определяется, как каждый модуль должен объявлять свои зависимости.  
Также - указано название всего проекта и какие модули он содержит.    

Каталог версий `/gradle/libs.versions.toml` - содержит список используемых библиотек и плагинов с их версями.  

Файл сборки проекта `/build.gradle.kts` - содержит на уровне проекта список плагинов из файла `libs.versions.toml`, которые могут быть использованы отдельными модулями. Если какой-то модуль пока не используется, у него будет параметр `false`.  

Файл сборки уровня плагина **app** `/app/build.gradle.kts` - содержит 
- список плагинов, используемых данным модулем `app`, 
- конфигурацию приложения, например версии проекта
- и список необходимых приложению зависимостей с отсылкой к файлу `/gradle/libs.versions.toml`.

## 023-Troubleshoot Android Studio projects

Если запущен старый проект, то следует обновлять зависимости gradle, особенно с предупреждением об обновлении при открытии проекта.

## 025-Solution Find the dependency

Библиотеки будут добавлены в файл `gradle/libs.versions.toml`

Большинство библиотек, расположенных на Github, описывают установку в файле Readme.md  

https://github.com/JakeWharton/timber  

    repositories {
        mavenCentral()
    }

    dependencies {
        implementation 'com.jakewharton.timber:timber:5.0.1'
    }

https://github.com/bumptech/glide

    repositories {
        google()
        mavenCentral()
    }

    dependencies {
        implementation 'com.github.bumptech.glide:glide:5.0.5'
    }

https://github.com/square/retrofit

    com.squareup.retrofit2:retrofit:3.0.0

Видимо, требование библиотек в секции repositories{} отсылает к секции repositories{} в файле `/settings.gradle.kts`.  
А dependencies{}, в свою очередь, ссылается на секцию dependencies{} в файле `/app/build.gradle.kts`.  

**Мой способ**

Вставил строку с `implementation` в файл `/app/build.gradle.kts` в секцию `dependencies {...}`. Строчка форматируется в вид

    implementation("com.github.bumptech.glide:glide:5.0.5")

При наведении курсора Android Studio просит переконвертировать в новый формат: "Replace with new library catalog declaration for glide, Alt+Shift+Enter". Жму. Получаю.

    implementation(libs.glide)

Нужные зависимости при этом автоматически прописываются в файле `/gradle/libs.versions.toml` .
