## 020-Manage Gradle build scripts

`settings.gradle.kts` - содержит список репозиториев, где Gradle будет искать зависимости, название приложения и список его модулей

`gradle/libs.versions.toml` - содержит список используемых библиотек с их версями и плагинов  

`build.gradle.kts` - содержит на уровне проекта список плагинов из файла libs.versions.toml, которые могут быть использованы отдельными модулями  

`app/build.gradle.kts` - содержит список плагинов используемых данным модулем `app`, версии проекта и список зависимостей с отсылкой к файлу `gradle/libs.versions.toml`

## 025-Solution Find the dependency

Библиотеки будут добавлены в файл `gradle/libs.versions.toml`

Большинство библиотек, расположенных на Github, описывают установку в файле Readme.md  

https://github.com/JakeWharton/timber  

    repositories {
        mavenCentral()
    }

    dependencies {
        implementation 'com.jakewharton.timber:timber:5.0.1'
    }

https://github.com/bumptech/glide

    repositories {
        google()
        mavenCentral()
    }

    dependencies {
        implementation 'com.github.bumptech.glide:glide:5.0.5'
    }

https://github.com/square/retrofit

    com.squareup.retrofit2:retrofit:3.0.0

**Мой способ**

Вставил строку с `implementation` в файл `app/build.gradle.kts` в секцию `dependencies {...}`.  
При наведении курсора Android Studio просит переконвертировать в новый формат. Жму. Получаю.

    implementation(libs.timber)

Нужные зависимости при этом автоматически прописываются в файле `gradle/libs.versions.toml`
