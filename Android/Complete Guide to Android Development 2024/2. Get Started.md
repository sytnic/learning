## 010-Create your first Android app

При установке нового проекта, указав название проекта, сразу будет предложена новая папка под него. Будет идти сохранение проекта в эту новую папку, которая будет создана автоматически. Создавать папку специально под проект можно, но не обязательно.    

После установки проекта, можно нажать на сплит (два экрана - для кода и для вида). Так будут добавлены ещё несколько файлов в проект.  

## 011-Explore Android Studio

Открыть окно логов

    View -> Tool Windows -> Logcat

Сочетания клавиш для поиска файлов и действий

    Ctrl+Shift+A - поиск классов и actions
    Ctrl+Shift+Space - подсказки, выбрать, как дописать код

## 012-Create virtual devices for testing

Добавление виртуальных девайсов для проверки своего приложения.  

Справа есть кнопка "Device Manager". Она же в меню "Tools - Device Manager". Далее через плюс "+" можно добавлять новые девайсы, пункт "Create Virtual Device".  

После выбора девайса, его запустить, нажав на запуск в списке девайсов. Далее запустить приложение на сборку "Run". Это установит приложение на виртуальный девайс. Теперь можно будет щёлкнуть по приложению на девайсе, чтоб запустить приложение.   

> Выключение девайса лучше осуществлять в списке девайсов по кнопке квадрата (Стоп). На изображении эрана девайса выключение может не работать.  

## 014-Set up debugging on a physical device

- Подключить телефон по USB  
  Возможно, понадобится драйвер на Windows для данного телефона

- Чтобы стать разработчиком на устройстве, нажать 7 раз на    
Build Number "Номер сборки" (в "О телефоне" - Версия).    
В Системе (System) появится значок "Параметры разработчика" (Developer Options)  

- В "Параметры разработчика" (Developer Options) включить отладку по USB (USB debugging)

Решение проблем:

- Menu VS Code -> Tools -> Troubleshoot Device Connections    

- https://developer.android.com/studio/run/device?hl=ru

## 015-Android command line tools

Командная строка есть в  
- SDK Tools
- SDK Build Tools
- SDK Platform Tools
- Android Emulator

CLI хранятся в `File > Project Structure > SDK Location` . 

Также есть инструмент командной строки ADB - Android Debug Bridge, выполняющий основные функции с девайсами, файлами и установкой программ APK.
